First Deploy blog with GitHub

Official Hexo doc EN - Deployment, Official Hexo doc CN - Deployment

  1. Create a new Public repo in GitHub to publish website online, with name format username.github.io, for example becoze.github.io

  2. Install deplorer: npm install hexo-deployer-git --save at blog root directory /HexoBlog

  3. Config deploy setting - use git to deploy: go to _config.yml almost at the end of code:

    • type: git (fixed)

    • repo: your repositories url (example of my repo)

      Here you can find the repositories url

    • branch: name of branch file pushed to (push to main for example)

  4. Deployment: run hexo deploy at blog root directory /HexoBlog

  5. Possible error (optional)

    ​ solution:

    • run git config --global user.email "you@example.com", your GitHub email address
    • run git config --global user.name "username", your GitHub username

5. Developed own Image hosting - “PicGo + GitHub”

Tutorial Bilibili 使用picgo+github搭建免费个人图床

  1. Create a new Public repo in GitHub with any name to store image, and add a README.md

  2. Obtain token: GitHub setting –> Developer setting –> Personal access tokens –> Generate new token

    • Note: any (for example “becozePictureHosting”)
    • Expiration: any (for example 60 days)
    • Select Scopes:
      • repo: MUST select all
      • all other: remain deselect

  3. Make sure to copy your personal access token now. You won’t be able to see it again!

  4. Download PicGo exe. file from GitHub at PicGo Release

    ​ PicGo-Setup-2.3.0-x64.exe for Windows

    ​ PicGo-Setup-2.3.0-x64.dmg for Mac iOS

x. Push source file to GitHub

Please always refer to Hexo official Doc EN Hexo - Deployment and CN Hexo - Deployment

  1. Create / Make-sure a .gitignore file at blog root directory /HexoBlog, contain following information

  2. Create a new Private repo in GitHub to store source file, for example /BlogSource

  3. Create a git repo: run git int

  4. Select the file need to be pushed: run git add .

  5. Push file to local: run git commit -m "First commit"

  6. Select GitHub branch to be upload: run git branch -M main

  7. Connect to GitHub online: run git remote add origin https://github.com/becoze/BlogSource.git

  8. Push file: run git push -u origin main

x.1. Recover all the file from GitHub

  1. Obtain GitHub repo url: go to your GitHub repositories, copy the HTTPS url under code. For example: https://github.com/becoze/BlogSource.git
  2. Clone file: run git clone https://github.com/becoze/BlogSource.git at your file. It will become your blog root file.
